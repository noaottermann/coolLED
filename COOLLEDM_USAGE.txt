CoolLEDM usage guide for this repository

This folder vendors the CoolLEDX driver code that is still useful for a CoolLEDM
device. The important parts are the BLE client, the hardware definitions, the command
builders, and the rendering helpers.

What to use

1. coolledx.client.Client
Connects to the device, discovers the hardware type from the BLE advertisement, and
sends commands.

Use it like this:

    from coolledx.client import Client

    async with Client(address="FF:00:00:04:1B:DF") as client:
        ...

For a CoolLEDM device, the client should detect the hardware after it connects. If you
already know the address, pass it directly. If you only know the device name, the
client can scan for it.

2. coolledx.hardware.CoolLEDM
Represents the CoolLEDM hardware type. In this vendored code it is created by the
client after connection; you normally do not instantiate it from the wrapper layer.

3. coolledx.commands
These are the high-level actions you send to the panel.

Use these for actual screen changes:

    SetText(text, ...)
    SetImage(filename, ...)
    SetAnimation(filename, ...)
    SetJT(filename, ...)

Use these for device settings or quick control:

    SetBrightness(level)
    SetSpeed(level)
    SetMode(mode)
    TurnOnOffApp(on=True/False)
    InvertDisplay(inverted=True/False)
    PowerDown()
    Initialize()
    StartupWithBatteryLevel(level)

Use SendRawData only for protocol debugging.

4. coolledx.render
These functions convert text, images, animations, or JT files into the byte payloads
that the device accepts. You usually do not call them directly unless you are building
new commands.

5. coolledx.utils.tweak_sign
This is the most useful example script if you want to send content from the command
line. It shows how the driver combines the client and the command classes.

What changed in this repository

The local wrapper in core/display.py now uses the command-based API correctly. It keeps
a local pixel buffer, converts that buffer into a temporary image, and sends the image
with SetImage. That means the wrapper is now a simple way to modify the display without
calling the upstream package as if it were a direct pixel-frame API.

Recommended workflow for a CoolLEDM panel

1. Connect with Client(address=...).
2. Choose a command class based on what you want to change.
3. For text, use SetText.
4. For a full-screen image, use SetImage with a file path.
5. For an animation, use SetAnimation with an animated file.
6. For exported CoolLED content, use SetJT.
7. Use SetBrightness, SetSpeed, SetMode, or TurnOnOffApp for panel behavior.
8. Disconnect when finished.

Notes

CoolLEDM uses the newer protocol path in this codebase. Some lower-level files such as
basic_protocol.py and advanced_protocol.py are still upstream scaffolding and are not
the main entry point for day-to-day screen changes.
